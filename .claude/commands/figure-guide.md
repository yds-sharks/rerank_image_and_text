# /figure-guide — 绘图审美指南与质量检查

提供科研绘图的审美规范指导，检查已有图表的质量，给出改进建议。

## 输入
用户提供：图片路径 或 绘图脚本路径 或 "全量检查"

## 审美规范体系

### 一、配色原则

#### 顶刊常用配色板
```python
# Nature 风格（沉稳专业）
nature_colors = ['#4E79A7', '#F28E2B', '#E15759', '#76B7B2', '#59A14F', '#EDC948']

# Science 风格（简约干净）  
science_colors = ['#0C5DA5', '#00B945', '#FF9500', '#FF2C00', '#845B97', '#474747']

# 柔和学术风（适合大面积填充）
soft_colors = ['#7FB3D3', '#F4A582', '#92C5DE', '#D6604D', '#B2ABD2', '#E08214']

# 对比强烈（适合少量类别）
contrast_colors = ['#E64B35', '#4DBBD5', '#00A087', '#3C5488', '#F39B7F', '#8491B4']
```

#### 配色禁忌
- 禁止彩虹色（rainbow colormap）：色盲不友好 + 不专业
- 禁止 `jet` colormap：用 `viridis`/`cividis`/`inferno` 替代
- 同图不超过 6 种颜色
- 相邻色彩区分度 > 30%（灰度打印也能分辨）

### 二、排版规范

#### 字体
- 正文/标签：Serif（Times/Computer Modern）或 Sans-serif（Helvetica/Arial）
- 全图字体统一，不混用
- 最小字号 ≥ 6pt（缩放到论文列宽后）

#### 尺寸
| 版式 | 宽度 | 常见场景 |
|------|------|---------|
| 单栏 | 3.3" (8.4cm) | 小图、子图 |
| 1.5栏 | 5.0" (12.7cm) | 中等图 |
| 双栏 | 6.75" (17.1cm) | 大图、架构图 |

#### 元素规范
- 坐标轴：有刻度线，无网格线（或极浅网格）
- 图例：放在图内空白处，不遮挡数据
- 标题：放在 caption 里，图内不加标题
- 子图标签：(a) (b) (c) 左上角，粗体
- 线宽：数据线 1-1.5pt，辅助线 0.5pt
- marker：大小适中（5-7pt），不同系列用不同形状

### 三、信息设计

#### 数据-墨水比（Data-Ink Ratio）
- 最大化数据墨水，最小化装饰墨水
- 删除不必要的边框、背景色、3D 效果
- 每个视觉元素都应传递信息

#### 视觉层次
1. 最重要的数据系列用最醒目的颜色/线型
2. Baseline 用灰色/虚线，Our method 用实色/实线
3. 用粗细、颜色深浅区分主次

#### 图表选型
| 数据类型 | 推荐图表 | 避免 |
|---------|---------|------|
| 趋势变化 | 折线图 | 柱状图 |
| 类别对比 | 柱状图 | 饼图 |
| 分布 | 箱线图/小提琴图 | 直方图（样本小时） |
| 相关性 | 散点图 | 表格 |
| 组成比例 | 堆叠柱状图 | 饼图（>5类） |
| 多维对比 | 雷达图/热力图 | 多线折线图（>6线） |

### 四、常见问题清单

- [ ] 字号是否足够大（缩放后 ≥ 6pt）？
- [ ] 配色是否色盲友好？
- [ ] 图例是否清晰且不遮挡数据？
- [ ] 坐标轴标签是否完整（含单位）？
- [ ] 是否有不必要的装饰元素？
- [ ] 灰度打印是否仍可辨识？
- [ ] 是否导出为矢量格式（PDF/SVG）？
- [ ] 子图标签 (a)(b)(c) 是否一致？

## 工作流程

1. 读取指定图片或脚本
2. 按审美规范逐项检查
3. 生成改进建议报告
4. 如果是脚本，直接给出修改后的代码

## 输出
- 终端输出审美评估报告和改进建议
- 如有修改建议，生成修改后的脚本
