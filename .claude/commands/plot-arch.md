# /plot-arch — 架构图/框架图绘制

绘制论文中的模型架构图、系统框架图、流程图。

## 输入
用户提供：架构描述（文字/草图/设计文档引用）、风格偏好

## 设计原则

### 布局
- 信息流向清晰：左→右 或 上→下，不混用
- 层次分明：用虚线框分组，标注每组功能
- 留白充足：组件间距 ≥ 组件高度的 30%
- 对齐严格：所有同层组件对齐，连线水平/垂直

### 配色规范
- 主色调 2-3 种，区分功能模块
- 推荐配色方案：
  - 蓝色系：输入/编码 `#4A90D9` → `#2C5F8A`
  - 绿色系：处理/计算 `#5BAA6B` → `#3D7A4A`
  - 橙色系：输出/解码 `#E8935A` → `#C4703A`
  - 灰色系：辅助/背景 `#E8E8E8` → `#B0B0B0`
- 同类模块同色，不同功能不同色
- 避免纯红/纯绿（色盲不友好）

### 标注
- 模块名：英文，粗体，居中
- 箭头上标注数据流名称（tensor shape / 数据类型）
- 关键操作用小号斜体标注（如 "softmax", "concat"）

## 技术路线

### 方案一：matplotlib + patches（推荐，可控性强）
使用 `matplotlib.patches` 绘制矩形、箭头、文字，完全代码控制。

### 方案二：drawsvg（复杂架构图）
使用 drawsvg 库生成 SVG 再转 PDF。

## 工作流程

1. 解析架构描述，确定组件和连接关系
2. 设计布局（网格对齐）
3. 选择配色方案
4. 生成绘图脚本
5. 输出 PDF 矢量图
6. 迭代调整（位置、大小、配色）

## 输出
- 脚本: `figures/code/figures_arch_{图名}.py`
- 图片: `figures/data/tmp/figures_arch_{图名}.pdf`
